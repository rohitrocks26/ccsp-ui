Scss best Practices

  • File Structure - Encourages common style, components and variable (inline with Atomic Design) Modular style management that facilitates Styleguide Driven Development
  • Styles.scss - Charset should be included Only directory files -dir.scss files should be imported Changes should not be made to this file Filename is always ‘styles’ as it is a compiled file containing multiple stylesheets
  • Vendors - All 3rd-party vendor tools (frameworks, libraries, helpers) should be separated into folders Only the 3rd-party master file for each vendor tool should be imported
  • Abstracts - Helper functions and non–output snippets only (font-face imports, reusable mixins, global variables, colors, etc.) One @import per line
  • For more details please follow this : Scss Best practices
